## Feature set (as shipped)
- **Capture → compare:** screenshot + link **OR just a link** (2026-07-16: I read the live page in the in-app browser — strictly richer than a screenshot: fee tabs, FAQ, published neighborhood rent averages, and the photo) → I extract fields, geocode, append to `data/apartments.ts`. Grid + Compare views, filters, sorts, status pipeline, ratings, Sheets export.
- **Distance ranking:** offline Bay Area ZIP/city table (429 ZIP + 106 city centroids) + cached Nominatim fallback; per-card chip + **Nearest sort (now the DEFAULT sort, 2026-07-10)**; runtime anchors in Settings. With no anchor set, Nearest degrades cleanly to insertion order (and no distance chips) until you pick one.
- **Owner / management contact** (per listing): company/owner, contact person, phone, email, website — shown on the card (name + tap-to-call/email/website rows below it) and the detail modal; editable in the form; exported. Safe `tel:`/`mailto:`/`https:` links; `tel:` encodes extensions (`;ext=`).
- **Your comments** (per listing): timestamped, add/delete; latest shown on the card below the flags, full thread in detail. Persisted as a user overlay and merged onto seed listings on reload (like ratings/status). A comment mentioning **"income restricted"** (any case, hyphen or space — `isIncomeRestrictedComment` in `lib/derive.ts`) renders highlighted **RED** on the card + detail thread: income qualification is a hard eligibility gate.
- **Find launcher** (header **Find** button → `FindModal`): pre-built Zillow / Apartments.com / Craigslist / FB Marketplace searches with the criteria baked in — `CRITERIA` in `data/searches.ts` = within ~15 mi of San Mateo (94402) or Redwood City (94063) · **$1,500–3,000** · 1+ bd / 1+ ba · short-term keywords. The criteria line (left of its **Copy** button) + every link sub derive from `CRITERIA`/`PRICE_LABEL`, so one edit retunes everything. **Craigslist links sort nearest-first** (`sort=dist`). Sites 403 automation → links open in the user's own browser.
- **Status pipeline + terminal dispositions** (ADR-011): New → Shortlist → Contacted → Toured → Applied → Rejected → **Leased** / **Ruled out** / **Gone**. Three distinct exits: **Gone** = off the market (neutral badge) · **Ruled out** = not available to you / didn't qualify (amber badge) · **Rejected** = you passed by preference (stays visible unless "Hide rejected"). Gone + Ruled out are **hidden by default**, each behind its own independent chip, and both are **reversible toggles** in DetailModal (always rendered, `.btn-mini.on` when lit, label flips to "Put back on list"; the modal stays open so undo is one click). The rule is the pure `toggleStatus(current, target)` in `components/helpers.tsx` — `current === target ? 'New' : target` — so un-mark RESETS to New (no prior-status memory; the toast discloses it) and clicking the other one cross-switches. Filters are session-only → no migration.
- **Lease goal = 6 months** (single-point). The lease flag is the **first flag after any red (risk) flags** on a card (lease fit is the #1 decision factor): 12-mo term → **red** "doesn't fit your 6 mo goal" (sorts into the risk group); no lease term stated → **amber** "confirm they'll do a 6 mo term"; true **month-to-month** → **green** "maximum flexibility". A safe localStorage migration bumps browsers still on the old 6/12 default → 6/6.
- **Red-flag gates + red-first ordering (2026-07-15):** two hand-set boolean gates render **red 'risk' flags** — `scamRisk` ("Possible scam — verify…") and `incomeRestricted` ("Income-restricted — you must income-qualify to rent; check the income cap…", for affordable/AMI properties like **a20/a24**). Both are Edit-form checkboxes and export columns ("Scam risk" / "Income-restricted"). `getFlags` ends with a **stable partition putting ALL risk flags first** (user rule: red on top); within the risks, precedence = scam > income-restricted > lease-conflict; among the rest the lease verdict still leads. Any risk flag also drives the red card top border (`signalLevel`).
- **Availability** (2026-07-07): a first-class `availability` status ('now' | 'unavailable' | 'unknown') behind `availableDate`. Card/detail/compare show a real date when stated, else **"Now"** (green) or **"Unavailable — ask"** (amber, rolling communities) + an amber warn to call. `leaseSummary()` renders friendly labels (Month-to-month / Flexible (1–N mo) / Short-term (N mo) / N mo / N–M mo). Amenity pill order: laundry, parking, balcony, gym.

  - **apartments.com 403s server-side image fetches** even with a browser UA + Referer → capture via the browser. **Zillow's CDN does NOT** — a plain `Invoke-WebRequest` works there, so don't over-engineer Zillow. Craigslist images fetch fine too (swap `_600x450` → `_1200x900`).
  - Full recipe + gotchas: **`~/.claude/skills/web-data-snapshot/references/browser-capture.md`** (+ `templates/imgcatch.js`). Load the `web-data-snapshot` skill.
- **Judgment calls that recur** (keep consistent): `marketRent` = the tightest *honest* comp — use a page's own **neighborhood** average over its city-wide one when the building sits in that neighborhood (city figures skew high from newer stock); set it **null** when there's no clean comp (furnished, income-restricted/AMI-priced, or a one-off like a tiny house) rather than inventing a "deal". Never pick a number to dodge/trigger a flag. Record dissenting reasoning in the notes instead of burying it.
